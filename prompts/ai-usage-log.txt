Prompt yang dijalankan dan akan di upload 
1. Prompt Class Diagram (Gemini)

Anda adalah arsitek sistem berpengalaman. Buat Class diagram UML untu sistem akademik perguruan tinggi dengan komponen berikut :
1. PMB Proses pendaftaran dan seleksi calon mahasiswa
2. KRS (Kartu Rencana Studi) Fitur bagi mahasiswa untuk memilih mata kuliah di setiap semester.
3. User sebagai base class (Mahasiswa, Dosen, Admin, Calon Mahasiswa)
4. Course degnan atribut kode, nama, SKS, prasyarat
5. Enrollment yanng hubungkan Mahasiswa dan Course
6. Schedule dengan hari, jam, ruangan
7. Grade Untuk Nilai mahasiswa. 

Terapkan prinsip SOLID. dan gunakan Composition untuk relasi kuat dan aggregation untuk relasi lemah, output dalam format Mermaid.

Validasi Diagram 
(Uload Image Diagram di Prompt)
Jadikan file terlampir sebagai satu-satunya sumber untuk dilakukan pengecekcekan apakah inheritance masuk akal
Pastikan tidak ada circular dependency pada diagram tersebut,
Verifikasi kardinalitas relasi (1-to-many, many-to-many) pastikan dengan baik.
